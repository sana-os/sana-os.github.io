なぜ SANA OS は存在するのか

問題が起きる時、それは誰かの悪意や能⼒不⾜ではなく、直⾯しているルールの「優先順位のズレ」が原因です。
犯人探しでは問題の原因がわからなくなってしまいます。

トラブルが発生した時、問題点ではなく犯人を捜して安心することを優先していませんか？

犯人を決めるのは簡単です。

「これは悪い判断だ。」
「あの人にはわかっていない。」
「どうせうまくいかない。」

犯人が見つかれば、不快感は消えますし、説明は完結したように感じます。でも、その下にある問題の構造は、見えないで残ったままになります。

例えば建物が倒壊しかけている時に、誰が建物を倒壊させたのか、設計者や工事責任者を責めて犯人を捜しても、倒壊しかけた建物はそのままです。犯人捜しは、倒壊しかけている建物のどの構造に原因があるかを確認してからでも遅くはないのではないでしょうか。

これは知性の問題ではありません。生まれ持った人間の認知の性質なのです。私たちは、不確実性を長く保持するようにはできていません。

SANA OSはひとつの目的のために存在します。それは、話し合いの前に、問題の起きている個所を特定すること。

問題をあなたの代わりに解決しません。誰が正しいかを決めたりはしません。ただ見えるようにするだけ。

相手の事情を理解したからと言って同意する必要はありません。でも互いの前提を知らずに話をしても、対話が成り立ちません。

何が原因かがわかれば
議論が、犯人捜しではなくなります。
原因がわかれば対応の話になります。
それが、本当の対話の始まりなのです。

なぜ機能するのか

対立はほとんどの場合、議題の問題ではない。
どの層が圧力を受けているかの問題だ。

人は常に、三層の圧力のもとで同時に動いています。これらが衝突したとき、脳は認知圧縮に入ります。複雑な思考が単純なラベルに崩れる。「間違い」「怠慢」「抵抗勢力」。議論はエスカレートするが、構造的な原因は見えないままです。

Layer A

生存層

食事・睡眠・安全・雇用。ここが脅かされると、他のすべてが止まる。

Layer B

物語・文化層

アイデンティティ・価値観・暗黙ルール・帰属感。ここへの挑戦は身体的な脅威として感じられる。

Layer C

制度・公式ルール層

法律・制度・KPI・契約。一晩で変えられる。ただし吸収はB層の速度でしか起きない。

核心：誰かの層を可視化すること—それだけで圧縮を中断するのに十分です。これらのツールが行うのはそれだけ。対立を解決するのではなく、対立の構造を可視化することで、当事者たちが自分で解決できるようにします。
